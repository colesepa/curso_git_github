git init -> cria/inicia um repositorio GIT
git status -> mostra os status de cada trabalho, com os arquivos a serem commitados
git commit -m "msg" -> faz commit dos arquivos alterados
git add -> adiciona arquivos para serem commitados 
git add . -> adiciona a pasta inteira para ser comitado
git add * -> adiciona todos os arquivos de uma pasta
git add *.txt -> adiciona todos dos arquivos com extensão txt
git log -> mostra o histório de alterações de todos arquivos
git diff . -> mostra todas as alterações trackeadar nos arquivos pelo git
git branch  -> lista as branchs existentes
git branch -d <nome> -> deleta a branch selecionado (nao pode estar ativa)
git checkout <nome> -> Muda para uma nova branch especificada, caso nao tenha o git mostrará um erro  
git checkout -b <nome> -> cria uma nova branch
git merge <nova_branch> -> Mescla os arquivos de uma outra branch
git pull origin <https...> -> Puxa os aquivos de uma repostiorio online para o repositorio local

git clone <https...> -> Clona um repositororio online para o repositório local
git push origin <nome_da_branch> -> Envia para o repositorio online aletrações feitas no repositorio loca.


Fluxo de Trablhos:

	1. Criar o diretorio do trablaho
	2. iniciar o git dentro do diretorio de trabalho (git init .)
	3. edita ou cria arquivos dentro do diretorio
	4. git status  - visualizar os arquivos alterados/adiconados/deletados
	5. git add <nome_do_arquivo> 
	6. git status
	7. git commit -m "msg" - adiciona as alterações dos aquivos
	8. git status
	9. volta para o passo 3.
